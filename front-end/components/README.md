# Componentes

Esta carpeta contiene todos los componentes reutilizables de la aplicación, organizados por dominio.

## Estructura

- `/admin`: Componentes específicos para la sección de administración
- `/dashboard`: Componentes para el dashboard de usuario
- `/organizer`: Componentes para organizadores de eventos
- `/public`: Componentes para las páginas públicas
- `/ui`: Componentes UI básicos y reutilizables

## Convenciones

- Cada componente debe tener su propio archivo
- Los componentes deben ser funcionales y usar TypeScript
- Usar props tipadas para todos los componentes
- Preferir composición sobre herencia
- Mantener los componentes pequeños y enfocados en una sola responsabilidad
